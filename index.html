What gets generated

After running /connect-integrate on a React + Vite project:

your-project/
├── server/
│   ├── index.js                    # Express server (port 4242)
│   └── routes/
│       ├── accounts.js             # Create accounts, onboarding links
│       ├── payments.js             # PaymentIntents with fee splitting
│       └── webhooks.js             # Stripe event handling
├── src/
│   ├── context/StripeContext.jsx    # Stripe Elements provider
│   └── components/
│       ├── CheckoutForm.jsx        # Payment form
│       └── ConnectOnboarding.jsx   # Operator/seller onboarding
├── .env                            # API keys (gitignored)
└── .connect-wizard-config.json     # Plugin state

Installation

Prerequisites

  • Claude Code installed and authenticated
  • Stripe account (free)
  • Connect enabled on your account
  • Test API keys ready — the plugin will ask for them during setup

Everything runs in Stripe's test mode, so no real money is involved.

Install from the Stripe internal marketplace recommended

  1. Open Claude Code
  2. Run /plugin
  3. Find the Stripe internal marketplace (update the marketplace if you don't see it)
  4. Search for connect-wizard and install it

Verify it works

terminal 
$ claude
> /connect-help          # see all commands

Changelog

0.2.1 2026-03-30

Published to the Stripe internal plugin marketplace

Connect Wizard is now available on the Stripe internal marketplace for Claude Code plugins. Install via /plugin — no more cloning or manual setup.

0.2.0 2026-03-24

Fixed — 10 friction points from external user testing

P0 — Blockers

  • False-positive Connect enablement check (connect-setup): Now uses stripe accounts create as a definitive test that catches missing ToS AND missing platform profile.

P1 — High

  • AskUserQuestion 4-option limit: Fixed 7 violations across 6 skill files where menus exceeded the hard 4-option max.
  • Webhook registration rejects localhost URLs: Replaced with stripe listen --forward-to for local dev.
  • Vite scaffolding fails in non-empty directories: Now scaffolds in /tmp and copies back. Pinned to vite@5.

P2 — Medium

  • CLI auto-install: Detects OS and attempts brew install (macOS) or apt install (Linux).
  • Account switching guidance: Shows existing profiles, explains --project-name.
  • Invalid CLI commands: Removed broken stripe accounts update for platform's own account.
Added — Risk management logic
  • New discovery question (Q5b) for fraud and risk management, conditional on business model.
  • Marketplace path: explains user is merchant of record, recommends Stripe Radar from day one.
  • Backend builder reads riskManagement config for Radar metadata and dispute handlers.
  • Config schema: added riskManagement { owner, radarEnabled, radarCustomRules, reason }.
